# update-requirements スキル

要件や設計方針を議論・更新するときのワークフロー。

## 手順

1. **現状の要件を確認** — CLAUDE.md の要件セクションを読む
2. **変更点を提案** — ユーザーと変更内容を議論する
3. **CLAUDE.md を更新** — 合意した内容を反映する
   - 現在の要件（v0.x）に追加・変更
   - 将来の要件チェックリストの更新
   - アーキテクチャへの影響があれば記載
4. **影響範囲の確認** — 既存コードへの変更が必要な場合はリストアップする

## CLAUDE.md の更新ルール

- バージョン番号は実装が完了した時点で上げる
- 検討中の要件は「将来の要件」に `[ ]` で記載する
- 廃止した要件は削除する（git履歴で追える）
