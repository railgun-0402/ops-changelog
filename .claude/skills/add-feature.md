# add-feature スキル

新機能を追加するときのワークフロー。

## 手順

1. **CLAUDE.md の「将来の要件」を確認** — 追加する機能が既に検討中か確認する
2. **設計を提案** — 実装前にユーザーと設計を合意する
   - 新しいフラグやサブコマンドが必要か
   - `internal/` のどのパッケージに追加するか
   - 既存の型・インターフェースを壊さないか
3. **実装** — `internal/` のロジック層から書き始め、`cmd/` をあとから繋ぐ
4. **CLAUDE.md を更新** — 要件・アーキテクチャセクションを最新の状態に保つ
5. **テスト追加** — `internal/` のロジックにユニットテストを書く

## 注意点

- フラグ名は既存のものと一貫性を保つ（`--repo`, `--service`, `--since` スタイル）
- GitHub API のレート制限を考慮する（ページネーション対応）
- エラーメッセージは障害対応者が読むことを想定して明確に書く
